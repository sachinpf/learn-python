# Graded Exercises

One file per Part 1 lesson. Each file has ~30 problems sorted by difficulty.

## How each file is organized

| Section | Count | What it's like |
|---|---|---|
| 🟢 Easy | ~10 | Almost verbatim from the lesson. Confidence-builders. |
| 🟡 Medium | ~10 | The lesson's ideas combined or twisted slightly. |
| 🔴 Hard | ~7 | Multi-step problems. You may need to plan before coding. |
| ⭐ Stretch | ~3 | Open-ended. Multiple correct answers. Push past comfort. |

Solutions are **collapsed by default**. Click "Show solution" only after you've genuinely tried. Trying-and-failing is where 90% of the learning happens.

## Tips

- **Type the code yourself.** Reading a solution is not the same as writing one.
- **Run every program.** If it errors, read the error message — that's Lesson 11's whole point.
- **Don't peek at the solution within 5 minutes.** If you're stuck, write down what you've tried, then peek.
- **After peeking, close the solution and try to write it again from scratch.** That's where it sticks.
