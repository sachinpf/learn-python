# Part 2: Practice & Curiosity

You've finished [Part 1](../01_foundations/README.md). You know the syntax. Now you need to **practice** until Python feels natural — and **explore** the curious questions that make programming actually interesting.

Part 2 has four parallel formats. Mix and match. No fixed order.

---

## The four formats

### 📝 [`exercises/`](exercises/) — Graded exercise sets per lesson

For each Part 1 lesson, a file with 20–40 problems sorted **easy → medium → hard → stretch**. Pure drill. Build fluency. Solutions are collapsed (`<details>` blocks) so you have to *choose* to peek.

Use these when you want to **strengthen a specific skill**. After Lesson 05? Open `exercises/exercises_05_conditions.md`.

### 🎯 [`packs/`](packs/) — Themed problem packs

Cross-topic problem packs that tell a story. Each pack has 8–12 problems progressing in difficulty. They mix concepts from multiple lessons.

Starter packs:
- Math Wizard Pack — primes, Fibonacci, factorials, secret codes
- Word Detective Pack — palindromes, anagrams, word puzzles
- Data Explorer Pack — CSV crunching, finding patterns
- Pattern Maker Pack — ASCII art, fractal-ish shapes
- Number Tricks Pack — guess my number, magic squares

Use these when you want to **apply what you know to a new problem** — not just drill it.

### 💡 [`did_you_know/`](did_you_know/) — Curiosity essays

Short reads (1–2 pages) that explain the **why** behind things you took for granted.

- Why does Python use indentation instead of `{ }`?
- What is a byte, really?
- How does `print()` actually put text on the screen?
- Why do lists start at 0, not 1?
- Why are bugs called "bugs"?

No exercises here — pure curiosity. Read whenever you want.

### 🧩 [`challenges/`](challenges/) — Mini-puzzles with progressive hints

One puzzle per file. Each puzzle has:
- The problem
- Hint 1 (gentle nudge — click to reveal)
- Hint 2 (more specific — click to reveal)
- Hint 3 (almost the answer — click to reveal)
- Solution (full code with explanation — click to reveal)
- Why this matters (one paragraph)

Aim for one challenge a day. They build the hardest skill of all: **problem-solving muscle**.
